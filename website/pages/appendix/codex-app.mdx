# Appendix A: Codex App

> **Type:** Reference | **Source:** OpenAI blog post "Introducing the Codex App"

This appendix summarizes the Codex App announced by OpenAI and how it complements this CLI-focused training.

## What It Is

Codex App is the desktop experience for Codex that combines:

- A coding agent workflow
- Access to ChatGPT context and conversation continuity
- A visual interface for prompts, files, and coding tasks

Use CLI when you want terminal-native workflows; use Codex App when you want a visual coding workspace tied to your ChatGPT workflow.

## Where It Fits in This Training

- **OpenAI Codex CLI**: best for shell-heavy workflows, automation, and repo-native iteration.
- **Codex App**: best for mixed chat + coding workflows and visual review loops.

The core prompting discipline from the course stays the same in both.

## What to Look For in the Official Demos

The official demos are useful if you watch them with a training lens:

- How the user frames tasks with clear outcomes and constraints.
- How Codex keeps iteration tight by focusing on one change at a time.
- How review and verification happen before accepting outputs.

In live delivery, this is the same pattern we teach in the CLI modules:

1. Define the target output.
2. Ask for a plan before broad changes.
3. Execute in small increments.
4. Verify results against requirements.
5. Document decisions in project files (`AGENTS.md`, notes, or reports).

## Suggested Add-on Exercise

After completing any exercise in this repo:

1. Re-run the same task in Codex App.
2. Compare iteration speed and review quality vs CLI.
3. Note where visual context helped and where terminal control was better.
