# Appendix B: MCP

> **Type:** Reference | **Prerequisites:** None

MCP (Model Context Protocol) lets OpenAI Codex use external tools (databases, APIs, issue trackers, filesystem servers).

---

## Core CLI Commands

```bash
codex mcp list
codex mcp get <name>
codex mcp add <name> --url https://example.com/mcp
codex mcp add <name> -- npx -y @modelcontextprotocol/server-filesystem /allowed/path
codex mcp remove <name>
codex mcp login <name>
codex mcp logout <name>
```

Use `codex mcp --help` and `codex mcp add --help` for exact options in your installed version.

---

## Configuration File

Codex CLI configuration lives in:

- `~/.codex/config.toml`

Example MCP entries:

```toml
[mcp_servers.filesystem]
command = "npx"
args = ["-y", "@modelcontextprotocol/server-filesystem", "/allowed/path"]

[mcp_servers.policy_db]
command = "npx"
args = ["-y", "@bytebase/dbhub", "--dsn", "${POLICY_DB_DSN}"]

[mcp_servers.jira]
url = "https://jira-mcp.company.com/mcp"
bearer_token_env_var = "JIRA_TOKEN"
```

Keep secrets in environment variables, not hardcoded in config files.

---

## Typical Insurance Use Cases

- Query policy/claims data from read-only databases.
- Pull Jira incidents for operational reporting.
- Fetch underwriting documents from controlled file shares.
- Trigger workflow systems through audited API tools.

---

## Security Guidance

- Only connect MCP servers from trusted providers.
- Prefer read-only credentials for production data.
- Keep approval policy conservative for high-impact tools.
- Review tool outputs before using numbers in client-facing deliverables.

---

## Key Takeaways

- MCP is the standard way to extend Codex with external systems.
- Start with `codex mcp add`, inspect with `codex mcp get`, and keep config in `~/.codex/config.toml`.
- Use environment variables for credentials and enforce least privilege.
