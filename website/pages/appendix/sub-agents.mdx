# Appendix C: Sub-agents

> **Type:** Reference | **Prerequisites:** None

OpenAI uses two related ideas:

- **Codex app/cloud workflows:** explicitly multi-agent, where multiple tasks can run in parallel.
- **Codex CLI sessions:** typically one active agent loop per session, with delegation done by splitting work into clear subtasks (or by running separate sessions).

For day-to-day work, the practical pattern is:

1. Ask Codex to propose a plan.
2. Split the work into independent subtasks.
3. Have Codex execute and summarize each subtask before final synthesis.

This keeps your main thread concise and reduces context overload.

---

## Recommended Delegation Pattern

Use prompts like:

```text
Analyze this portfolio in parallel:
1) data quality checks,
2) trend analysis by region,
3) outlier claim review.
Then synthesize findings into one executive summary.
```

Then follow with:

```text
Before implementation, list assumptions and risks for each subtask.
```

---

## Model Guidance

Prefer Codex-targeted model names available in your environment (for example `gpt-5.3-codex`).

If a selected model is unavailable, retry with another model your account allows.

---

## Practical Guardrails

- Keep each delegated subtask outcome-focused (one output per subtask).
- Require explicit assumptions and data limitations in every subtask summary.
- Save intermediate outputs to files (`analysis_region.md`, `outliers.csv`, etc.).
- For high-impact actions, keep approval policy conservative (`-a on-request`).

---

## Example Workflow (Insurance)

```text
Task A: Validate claims dataset quality and report missing fields.
Task B: Compute loss-ratio trends by country and product line.
Task C: Flag top 20 anomalous claims and explain why.
Final: Merge A/B/C into a 1-page management brief with 5 actions.
```

---

## Key Takeaways

- In Codex app, multi-agent execution is a native pattern; in CLI, treat delegation as structured task decomposition.
- Delegation improves speed and clarity for complex work when subtask boundaries are explicit.
- Use Codex-available model names and verify capabilities against your installed Codex version.
