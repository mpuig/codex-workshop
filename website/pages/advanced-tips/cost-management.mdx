# Cost & Usage Management

> **Type:** Reference | **Prerequisites:** None

## How OpenAI Codex Usage Works

OpenAI Codex is free to install. Usage costs depend on how your organization authenticates:

- **ChatGPT login (`codex login`)**: usage follows your ChatGPT plan and account limits.
- **API key login (`OPENAI_API_KEY`)**: usage is billed by API token consumption in the OpenAI platform.

For training, either approach works.

---

## Model Choice

Use model names from the GPT family in Codex (for example `gpt-5.3` where available in your account/org).

Example:

```bash
codex -m gpt-5.3 "Draft an underwriting summary from q3_claims.csv"
```

If a model is not available for your account, Codex will show an error and you can retry with another allowed model.

---

## Token Basics

A **token** is the unit used for model context and billing.

- Roughly ~4 characters per token (rule of thumb)
- Large files and long conversations use more tokens
- Re-reading the same long context repeatedly increases usage

---

## Practical Cost Controls

1. Keep tasks scoped: ask for one concrete output per prompt.
2. Break large work into phases: research -> plan -> implement.
3. Save outputs to files early so you can restart with a clean session when needed.
4. Use planning-first prompts before big edits ("Propose a plan first, do not edit yet").
5. For automation-heavy workflows, prefer API-key mode so usage is trackable in the OpenAI dashboard.

---

## Checking Usage

- ChatGPT-authenticated usage: check limits from your ChatGPT account settings.
- API-key usage: check [OpenAI Usage](https://platform.openai.com/usage).
- Auth state in CLI:

```bash
codex login status
```

---

## Key Takeaways

- Avoid provider-specific model names from other ecosystems (from other model families) in Codex training.
- Use GPT-family model names (for example `gpt-5.3`) when available in your environment.
- Prompt quality and session discipline are the biggest levers for controlling usage.
